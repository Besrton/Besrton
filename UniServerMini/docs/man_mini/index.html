PHP MiniServer

Introduction

Welcome to The PHP MiniServer and this documentation. The PHP MiniServer is a free and open source Windows PHP server suite. Comprising of PHP with its built-in web server an smpt mail client and an easy to use user interface utility.

These components are integrated to provide a system suitable for both beginning webmasters and experienced developers. It is ideal for development use and small enough to carry on a USB memory stick, or even a camera's flash drive.

We strongly suggest that you check over the points on this page as they provide important information that may help you avoid common problems. After that, if you are eager to get started, you can jump straight to the quick start guide. It will have you up and running in seconds.

Note: This documentation is applicable to both MiniServer and MicroServer the MicroServer is a cut down PHP MiniServer specifically ICU is not included.

Background

Starting with PHP 5.4 is the addition of a built-in basic web server. Advantage of such a server it provides a self contained PHP development environment removing the need to use and configure third party servers such as Apache or IIS. Running the built-in server requires familiarity with the command line interface as follows.

Note: The PHP MiniServer user interface utility provides a graphical user interface removing the need to remember command line sequences or creating batch files.

Starting server:

The PHP built-in server is started with the command-line option -S followed by a host address (either in numeric or alphabetic) and a port number delimited by a colon. The server will listen on the given address and port. Press CTRL-C to stop server.

Host address:
  • localhost – Server is accessible only from local machine
  • 0.0.0.0 – Server is accessible from any interface wired or wireless.
    • Providing local and Internet access
 
php -S localhost:8080

<full path>\php -S localhost:8080
Specifying PHP configuration file:

The PHP built-in server uses default file php.ini for its configuration options. You can target a different configuration file with the command-line option -c followed by the file to be used for example php2.ini.

 
php -S localhost:8080 -c php2.ini

<full path>\php -S localhost:8080 -c <full path>\php2.ini
Specifying Document Root:

The document root is specified using the -t option. If omitted, the current working directory is used.

Note: Directory Indexes:
  • If a request is made for a directory, the web server will return
    index.html or index.php if they exist in the directory.
  • If these files do not exist a not found message is returned.
 
php -S localhost:8080 -t docroot

<full path>\php -S localhost:8080 -t <full path>\docroot
Specifying Router Script:

Apache has the ability to redirect all incoming requests using URL rewriting. The PHP built-in server emulates this functionality using a router script. A router script is run on every request to the web server.

 
php -S localhost:8080 -t docroot router.php

php -S localhost:8080 -t docroot <full path>\router.php

The output of a router script is parsed and executed by the server. If a router script returns false, the request is handled in the default manner. Otherwise, the script returns an appropriate response to the client.

Router Script example:

The following example checks incoming requests. If the requested resource exists it is returned. If the resource does not exist the user returned the index.php page.

 
 //Check resource exists
 if (file_exists(__DIR__ . '/' . $_SERVER['REQUEST_URI'])){
   return false;             // Exists serve resource as requested.
 } else {                    // Not found
   include_once 'index.php'; // Serve index page
 }
Summary

The command line has the following format when using all command line parameters :

<full path>\php -S localhost:8080 -c <full path>\php2.ini -t <full path>\docroot <full path>\router.php

Installation

The PHP MiniServer does not have a typical program installation. Instead, just execute the downloaded file (UniServerMini_x_x_x.exe or UniServerMicro_x_x_x.exe).

The installation extraction defaults to creating "C:\UniServerMini" or "C:\UniServerMicro".
Folder structure for UniServerMini is shown on the right. UniServerMicro folder structure is similar.
Once extracted, the server is ready to run.

Installation Restriction

While it may seem that by this point you have successfully completed the installation, there is one aspect that needs to be validated. This is especially true if you have been unable to start the server.

Server Paths

The installation extraction defaults to creating "C:\UniServerMini" or "C:\UniServerMicro". The documentation examples assume this default. You can install The Uniform Server PHP MiniServer in nearly any location, with the exception that the path to folder UniServerMini and UniServerMicro must not contain spaces. Do not use "C:\Program Files", for example (which has other problems as well, such as restricted permissions and system restore inconsistencies). We recommend installing to the drive root, such as "C:\" or "D:\" for simplicity, but other locations, such as "C:\test1\appserver" work just as well.

Note: The space character restriction is generic and applies to WAMP stacks as well. Many applications hosted on servers do not tolerate spaces in paths.

If the current installation is on a faulty path, the situation can eaily be resolved by just moving the UniServerMini or UniServerMicro directory to a proper location. You do not have to delete the directory and reinstall. For example, just rename "D:\bad path\UniServerMini" to "D:\good_loc\UniServerMini".

Other Issues

This section covers the basics a new user should be aware of.

VC9 Libraries

If you are using Windows XP as the OS platform for the PHP MiniServer, please be aware that you may need to install the Microsoft Visual C++ 2008 runtime library. This can be found at Microsoft Visual C++ 2008 Service Pack 1 Redistributable Package ATL Security Update. Be sure to select the package appropriate for your OS and hardware (x86, x64 or IA64).

If you do not have the VC9 library, when you attempt to start the servers, you will get a runtime error popup.

Windows 7, Vista and UAC

The startup executables have been marked as requiring Administrator privilege. When you start these programs, you must authorize this access in order to continue.

Firewalls

When you initially start the server, your firewall will sense the programs' requests for IP access, and (typically) will require your authorization to change the security. If you have a third-party firewall activated (Symantic, ZoneAlarm, etc.), be sure to set the port access as required, since it may not display a message.
